# 🏨 Сервис поиска отелей с "умным" поиском

Этот проект представляет собой систему бронирования отелей на **Django**, дополненную API на **FastAPI** для работы с нейросетевыми профилями.

## 🚀 Основные возможности
- **Поиск по названию:** Работает в любом регистре (найдет "Отель", если введено "от").
- **Фильтрация:** Удобный выбор городов прямо в шаблоне.
- **Генерация данных:** Команда `fill_db` наполняет базу отелями в разных городах.
- **Docker Ready:** Запуск всей инфраструктуры одной командой.

## 🛠 Запуск проекта через Docker
1. Убедитесь, что у вас установлен Docker и Docker Compose.
2. Склонируйте репозиторий.
```bash
   git clone https://github.com/t84625hsx9-del/hotels.git
   cd project
   ```
3. Запустите сборку:
```bash
docker-compose up --build
```
4. Django будет доступен по адресу: `http://localhost:8000`
5. FastAPI будет доступен по адресу: `http://localhost:8001`

## 🧪 Запуск проекта через Python(1 способ)
- **Наполнение базы данными:**
  `python3 manage.py fill_db`
- **Запуск Django:**
  `python3 manage.py runserver`
- **Запуск FastAPI:**
  `python3 -m uvicorn fastapi_app:app --port 8001`
- **Для автоматической отмены просроченных броней и имитации возврата средств запустите:**
`python3 manage.py process_refunds`

## 🧪 Запуск проекта через Docker (Рекомендуемый способ)
1. **Запустите контейнеры:**
   `docker-compose up -d --build`
2. **Для статичных файлов(таких как bootstrap.min.css):**
`docker-compose exec web python manage.py collectstatic --noinput`
3. **Создайте структуру таблиц:**
   `docker-compose exec web python manage.py migrate`
4. **Наполните базу отелями:**
   `docker-compose exec web python manage.py fill_db`
5. **Создайте администратора:**
   `docker-compose exec web python manage.py createsuperuser`
6. **Автоматическая отмена просроченных броней:**
   `docker-compose exec web python manage.py process_refunds`

## ⚙️ Особенности поиска
В проекте использована база данных **PostgreSQL**, что позволило реализовать быстрый регистронезависимый поиск по кириллице с использованием `icontains`. Это гарантирует мгновенное нахождение отелей независимо от того, в каком регистре введен запрос.

## 💎 Почему этот проект уникален?

1. **Гибридная архитектура (Django + FastAPI):** 
   Я совместил фреймворк Django (админка, ORM, аутентификация) и скорость FastAPI. Это позволяет выносить тяжелые операции (например, работу с нейросетями или профилями пользователей) в отдельный микросервис, не нагружая основной интерфейс.

2. **"Умный" поиск на "глухой" базе:** 
   Реализована кастомная логика поиска для SQLite, которая обходит системные ограничения базы данных при работе с кириллицей. Поиск находит данные независимо от регистра и сложности названий (включая кавычки и аббревиатуры), что делает UX максимально близким к коммерческим сервисам.

3. **Синтетическая память нейросети:** 
   В проекте заложена база для создания персонализированного опыта. Функция `get_user_ai_profile` имитирует память нейросети, собирая историю бронирований и предпочтений гостя, что позволяет в будущем внедрить рекомендательную систему на базе ИИ.

4. **Полная автоматизация (Seed-логика):** 
   Система поставляется с продвинутым генератором данных на базе Faker. За одну секунду база наполняется реалистичными отелями с привязкой городов к адресам и описаниям, что позволяет мгновенно тестировать фильтрацию и поиск в условиях, близких к реальным.
